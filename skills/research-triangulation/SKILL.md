# 研究三角验证

分析结论的质量上限由数据来源的质量决定，而不是分析方法的复杂度。

## 触发时机

- `stock-report` Step 1 数据收集完成后，进入分析前
- 发现不同来源的数据相互矛盾时
- 引用任何驱动结论的关键数据点之前
- 对某个数字感到不确定，但不知道该不该信时

---

## A 股来源可靠性层级

```
L1 — 法律强制披露（可信度最高）
     证监会/交易所公告 · 经审计年度报告 · 季度报告
     说谎有刑事后果，造假成本极高

L2 — 经核实的公司信息
     投资者关系官方回复 · 业绩说明会记录 · 招股说明书
     公司主动披露，有美化动机，但受监管约束

L3 — 独立第三方
     无投行业务关联的独立研究机构 · 做空报告 · 学术论文
     无利益冲突，覆盖范围有限，有时有相反方向的偏见

L4 — 卖方研究（结论需打折）
     券商研报 · 分析师评级
     ⚠ 结构性偏多陷阱：券商依赖上市公司投行业务，
       给负面评级会影响商业关系
     → 事实数据引用可信，方向性结论打七折

L5 — 主流财经媒体
     财新 · 经济观察报 · 21世纪经济报道
     需向上追溯原始来源，媒体本身不是一手信息

L6 — 不可单独引用
     股吧 · 雪球 · 社交媒体 · 匿名分析
     只能用于发现线索，严禁作为结论依据
```

---

## 三源验证规则

**任何驱动结论的关键数据点，强结论需 ≥3 个独立来源；次要数据需 ≥2 个。**

**"独立"的定义（满足所有条件才算独立）：**
- 不是同一份原始数据的多次转引（3家媒体引用同一份年报 = 1个来源）
- 不是同一机构的不同报告
- 不是同一时间窗口内的重复确认

**无法完成三角验证时的强制处理：**
1. 在输出中明确标注「单一来源」或「双源待验证」
2. 将该结论的置信度降至「主观判断」档（见 `analytical-rigor`）
3. 主动搜索反向证据，而非继续堆积正向证据

---

## 矛盾信息裁决规则

不同来源数据相互矛盾时，按以下顺序处理：

1. **层级胜出**：L1 年报数据与 L4 券商报告矛盾 → 以 L1 为准，记录差异
2. **时间胜出**：同层级来源中，较新的披露优先
3. **具体胜出**：精确数字比定性描述更可信
4. **矛盾本身是信号**：记录矛盾，追问"为什么会有这个差异"——差异往往比数据本身更有价值

**红旗情况**：L1 与 L2 之间存在矛盾 → 停止分析，在结论中明确标注为重大风险信号。

---

## 输出前检查

- [ ] 核心财务数据均来自 L1 或 L2
- [ ] 没有使用 L6 来源支撑任何结论
- [ ] 识别了所有 L4 来源的潜在利益冲突
- [ ] 所有矛盾数据已记录并说明裁决理由
- [ ] 无法三角验证的数据已降级标注
